## Objetivo
Garantir a qualidade e estabilidade do software através de testes automatizados.

## Checklist
- Testes unitários cobrem casos de sucesso e erro?
- Testes de integração validam contratos entre camadas?
- Há cobertura para fluxos críticos (happy path e edge cases)?
- O código de teste é limpo e independente?
- O build passa com os novos testes?
